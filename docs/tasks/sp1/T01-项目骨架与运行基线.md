## 任务目标

建立 MVP 的工程骨架与运行基线，保证后续 Rust、Tauri、Vue、SQLite、移动端页面和局域网服务可以在统一目录与配置结构下推进。

## 任务范围

- 初始化 Tauri 2 桌面工程与 Vue 3 + TypeScript + Vite 前端工程
- 明确 Rust 模块边界，例如 `clipboard`、`http`、`auth`、`persistence`、`app_state`
- 建立桌面端与移动端页面入口组织方式
- 配置 SQLite 文件位置、应用配置目录、日志初始化
- 明确环境配置项，例如监听端口、轮询频率、最大文本长度、token 有效期

## 产出物

- 可运行的项目目录结构
- Tauri 命令与事件入口骨架
- Rust 应用状态对象与模块骨架
- 前端路由或页面入口骨架
- 基础配置与默认值定义

## 完成标准

- 项目目录能承载桌面 UI、移动 H5、Rust 服务和数据库模块
- 模块命名和边界足够清晰，后续任务不需要大改目录
- 有明确的全局配置入口和默认值
- 能支持后续接入 SQLite、HTTP 服务和剪贴板轮询

## 实施提示

- 优先先定模块边界，再写页面骨架
- 移动端 H5 与桌面端 UI 可以共用前端工程，但要避免状态混杂
- 不在本任务实现完整业务逻辑
