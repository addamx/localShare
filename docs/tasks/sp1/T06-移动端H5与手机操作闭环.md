## 任务目标

实现手机端轻量 H5 页面，覆盖历史浏览、文本提交、历史激活、会话状态提示和刷新动作，完成移动侧闭环。

## 任务范围

- 顶部状态区展示设备名、在线状态、会话有效期
- 文本输入区支持提交新文本与提交反馈
- 历史列表区支持浏览、搜索、仅看置顶、刷新
- 点击历史项后直接激活到桌面剪贴板
- 处理 token 过期、鉴权失败、提交失败、空内容校验

## 产出物

- 移动端页面与组件
- 手机端 API 调用封装
- 会话初始化与 URL token 读取逻辑
- SSE 或刷新驱动的历史同步逻辑

## 完成标准

- 扫码打开后能快速加载历史
- 提交文本后桌面端可见且移动端反馈明确
- 点击历史项可直接触发激活
- 会话失效时能明确提示重新扫码

## 实施提示

- token 仅保存在 session 级上下文，不做长期持久化
- 写入成功后优先刷新历史并回到顶部
- 首屏要避免空白等待过长
