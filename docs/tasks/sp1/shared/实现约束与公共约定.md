# SP1 实现约束与公共约定

## 架构与职责约定

- 前端负责展示与交互，不直接写 SQLite
- Rust 负责剪贴板 I/O、会话认证、HTTP 服务、数据持久化和日志
- 剪贴板监听链路与 HTTP 服务解耦
- 事件推送只用于界面刷新，不作为最终一致性的唯一依据

## 数据与接口约定

- 统一数据源为 SQLite
- 至少包含 `sessions`、`clipboard_items`、`audit_logs`、`app_settings` 等核心表
- API 返回统一采用 `ok/data/error/ts` 结构
- 写接口统一走服务端校验和鉴权
- `clipboard_items` 需要支持搜索、置顶过滤、分页或增量刷新

## 安全约定

- 默认只开放局域网访问
- token 明文不落库，至少保存哈希
- 写操作必须鉴权，包括提交、激活、删除、置顶和 token 轮换
- 请求日志与审计日志默认脱敏 token 与完整内容
- 要限制文本大小和请求频率

## 运行与体验约定

- 端口冲突时需要可恢复处理
- 服务不可用时，界面应明确展示原因
- token 轮换后需要立即刷新二维码和会话展示
- 手机首次进入应尽快拉取历史，避免空白首屏
- 激活和提交动作要有明确成功或失败反馈

## 任务引用方式

各任务默认遵守本文件中的实现基线；若有差异，需在任务文件中单独标注。
