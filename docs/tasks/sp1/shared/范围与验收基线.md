# SP1 范围与验收基线

## 目标范围

SP1 只覆盖 MVP 主链路：

- 桌面端持续监听纯文本剪贴板并保存历史
- 桌面端展示历史、搜索、置顶、删除、清空、激活
- 桌面端展示局域网访问入口、服务状态、二维码和会话状态
- 手机端通过扫码访问 H5 页面
- 手机端查看历史、提交文本、激活历史项
- token 具备失效与轮换能力
- 服务重启后历史可恢复

## 明确不包含

- 账号体系与公网访问
- 图片、文件、富文本
- 多桌面设备协同
- 标签、分组、导出、批量编辑
- 复杂运维后台

## 关键产品规则

- 仅支持 `text/plain`
- 手机访问必须依赖短期会话 token
- 空内容不可提交
- 历史列表优先保证可用性和检索效率
- 用户需要明确感知同步、失败和会话状态

## SP1 验收口径

- 桌面端复制纯文本后，可自动入库并在界面可见
- 手机扫码后，可查看历史列表
- 手机提交纯文本后，桌面端可看到并直接使用
- 任一端激活历史项后，系统剪贴板更新成功
- 搜索、置顶、删除、清空等基础管理能力可用
- token 过期后旧链接不可继续写入或读取敏感数据
