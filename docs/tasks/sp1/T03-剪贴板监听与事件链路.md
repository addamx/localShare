## 任务目标

实现桌面端文本剪贴板监听、去重入库、当前项维护和前端刷新事件广播，打通 MVP 的核心数据生产链路。

## 任务范围

- 基于 Tauri clipboard 能力轮询系统剪贴板
- 仅处理纯文本内容，忽略非文本内容
- 对空值、超长文本和短时间重复内容做过滤或去重
- 新内容入库后更新 `is_current`
- 向桌面端 UI 和 SSE 订阅方广播刷新事件

## 产出物

- Clipboard Service
- 去重与内容哈希逻辑
- 当前剪贴板状态更新逻辑
- 前端事件广播约定

## 完成标准

- 用户复制纯文本后，可自动生成历史记录
- 重复内容不会在短时间窗口内无意义刷屏
- 当前项标记准确
- 新记录产生后，桌面端和移动端订阅方可收到刷新信号

## 实施提示

- 轮询频率需要可配置，默认采用温和频率
- 广播事件只传递刷新信号和必要元数据，避免泄露完整文本
- 监听异常不应导致服务整体崩溃
