## 任务目标

建立符合设计文档的数据表、索引、迁移与仓储接口，作为剪贴板监听、局域网 API、桌面端和移动端的统一数据基础。

## 任务范围

- 创建 `devices`、`sessions`、`clipboard_items`、`audit_logs`、`app_settings` 表
- 建立去重、列表排序、会话校验、审计查询等必要索引
- 封装仓储接口，覆盖历史读写、会话读写、审计写入、配置读取
- 统一逻辑删除、当前剪贴板唯一性、置顶排序等数据规则
- 预留迁移入口，支持后续 schema 升级

## 产出物

- SQLite 初始化脚本或迁移文件
- Rust persistence layer 与 repository 接口
- 数据模型定义与字段校验逻辑
- 基础查询与事务边界

## 完成标准

- 历史、会话、审计、配置四类数据均可稳定读写
- 当前剪贴板状态和逻辑删除规则可落地
- 后续模块可不关心 SQL 细节，直接依赖统一仓储接口
- schema 与技术文档字段保持一致

## 实施提示

- `clipboard_items` 建议优先把读模型和写模型拆开考虑
- `sessions` 只保存 token 哈希和状态，不明文存储 token
- 清空历史可作为逻辑删除批量操作实现
